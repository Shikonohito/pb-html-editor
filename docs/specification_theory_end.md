# Спецификация интерактивного теоретического модуля

Данная спецификация описывает структуру, логику и стилизацию шаблона интерактивной веб-страницы для отображения учебных материалов по программированию (на примере Python). Все компоненты находятся в директории `example_theory`.

## 1. Обзор проекта
Проект представляет собой интерактивную страницу на чистом HTML, CSS и JavaScript. Она предназначена для предоставления теоретического материала с интерактивными элементами: боковой навигацией, сворачиваемыми блоками контента, тестами с моментальной клиентской проверкой и встроенной песочницей для выполнения кода на Python прямо в браузере.

### Файловая структура
*   `index.html` — разметка страницы, структура контента, SVG-спрайты и подключение внешних библиотек через CDN.
*   `style.css` — стили, CSS-переменные (цвета, размеры, шрифты), адаптивный дизайн (медиа-запросы).
*   `script.js` — логика интерфейса (сайдбар, оглавление), инициализация редактора кода и интерпретатора Python, а также логика проверки тестов.

## 2. Макет и навигация (Layout)

Макет разделен на боковую панель (Sidebar) и основную область контента (Content Wrapper), занимающую оставшееся пространство (flexbox). Используется 100vh контейнер без скролла на `body`.

### Боковая панель (Sidebar)
*   **Состав:**
    *   Кнопка-гамбургер для переключения состояния панели (на десктопе).
    *   Логотип в виде круга.
    *   Блок копирайта/информации о создателе.
    *   Оглавление (TOC - Table of Contents).
*   **Оглавление:** Содержит ссылки-якоря на разделы контента (`<section id="...">`). Активный раздел подсвечивается маркером `◀`. Текст ссылок обрезается с градиентом с помощью `mask-image`.
*   **Адаптивность:**
    *   **Десктоп (>1024px):** Панель может быть развернута (`18rem`) или свернута в узкую полосу (`3.05rem`), где скрывается текст и маркеры, оставляя только узкую панель для экономии места.
    *   **Планшеты и Мобильные (<=1024px):** Панель по умолчанию скрыта за левым краем экрана. Вызывается по клику на плавающую кнопку (FAB) в правом нижнем углу или кнопку в шапке контента. Выезжает поверх контента вместе с полупрозрачным фоном (`#overlay`). При клике на ссылку оглавления панель автоматически скрывается.

### Область контента (Content Wrapper)
*   Скроллится независимо от сайдбара.
*   **Шапка контента (`.content-top`):** Фиксируется при прокрутке (sticky). Содержит главный заголовок текущего урока.
*   **Основной контент (`.main-content`):** Ограничен по максимальной ширине (1024px), центрирован.
*   Состоит из логических секций (`.lesson-section`), разделенных визуально `hr.section-divider`.

### Скроллинг и Intersection Observer
*   Переходы по якорным ссылкам в оглавлении перехватываются JS. Рассчитывается позиция для плавного скролла (`behavior: 'smooth'`), так, чтобы заголовок целевой секции оказался не прижатым к верхнему краю, а находился в верхней четверти экрана.
*   Используется API `IntersectionObserver` с `rootMargin: '-30% 0px -50% 0px'` для автоматического определения раздела, который сейчас читает пользователь, и обновления активного класса в оглавлении.

## 3. Оформление и стилизация (CSS)

*   **Переменные (CSS Custom Properties):** Все ключевые цвета, размеры и шрифты вынесены в `:root`.
    *   Основной цвет фона боковой панели/шапок: `#0a5a95` (синий).
    *   Акцентный цвет текста/выделения: `#fbbf24` (золотисто-желтый).
    *   Цвета обратной связи: Правильно - `#28A745`, Неправильно - `#DC3545`.
*   **Шрифты:** Подключаются из Google Fonts. `Inter` для обычного текста, `JetBrains Mono` для моноширинного (блок кода, песочница).
*   **Сброс стилей:** Box-sizing: border-box.
*   **Доступность (A11y):** Кастомный `outline` при навигации с клавиатуры (`*:focus-visible`).
*   **Скроллбар:** Стилизован в сайдбаре (с помощью вендорных префиксов webkit и свойства `scrollbar-width` для Firefox).

## 4. Контент и специальные блоки

Текст разбит на секции. Секции не могут быть вложены друг в друга. Внутри секций могут быть параграфы и специальные блоки.

### Специальные блоки (`.special-block`)
Блоки акцентируют внимание на определенной части контента.
*   **Структура:**
    *   **Контейнер (`.special-block`):** Задается рамка (border) цвета основного фона.
    *   **Шапка (`.block-header`):** Заголовок с эмодзи-индикатором и кнопка-стрелка сворачивания/разворачивания (`.toggle-block`).
    *   **Обертка тела (`.block-body-wrapper`):** Создается динамически через JS и использует CSS Grid (`grid-template-rows: 0fr` <-> `1fr`) для плавной анимации скрытия/показа контента блока.
    *   **Тело (`.block-body`):** Внутренний контент блока.
*   **Вложенность:** Допускается не более 2 уровней вложенности блоков друг в друга.
*   **Типизация (Эмодзи в заголовках):**
    *   📚 - Определение термина.
    *   ⚙️ - Объяснение алгоритма/кода/идея.
    *   ⚠️ - Предупреждение.
    *   💡 - Подсказка.
    *   ✅ / ❌ - Верный / неверный подход.
    *   ❓ - Тест / Вопрос.
    *   💻 - Песочница.

## 5. Блоки с кодом (Code Blocks)

Отображают сниппеты кода.

### Код только для чтения
*   Контейнер `.code-container` с темным фоном (`#1e1f22`).
*   Содержит `<pre><code>`.
*   **Копирование:** Кнопка `.copy-btn` в правом верхнем углу (отображается при наведении). Использует `navigator.clipboard.writeText`. При успешном копировании иконка меняется на галочку (через SVG `<use>`) на 3 секунды.

### Исполняемый код (`.runnable-code`)
*   Расширяет обычный код кнопкой "▶ Запустить" и скрытым блоком вывода (`.sandbox-output-wrapper`).
*   Код из HTML отправляется на выполнение в интерпретатор Pyodide, результат выводится под блоком кода. Поддерживает работу с `print()` (описано ниже).

## 6. Интерактивные тесты

Проверка тестов реализована на стороне клиента с использованием `data-` атрибутов. Это позволяет не зависеть от бекенда.

### Одиночный выбор (Радио-кнопки)
*   Структура: `<div class="radio-group">` с вложенными `<input type="radio">`.
*   Логика:
    *   Правильный ответ задан атрибутом `data-correct="true"`.
    *   Текст объяснения задан в `data-feedback`.
    *   При выборе генерируется блок `.inline-feedback` под выбранным вариантом. Применяется класс `.feedback-correct` или `.feedback-wrong`.
    *   Объяснения других вариантов в этой же группе сворачиваются.

### Множественный выбор (Чекбоксы)
*   Структура: `<div class="checkbox-group">` с вложенными `<input type="checkbox">`.
*   Логика: Аналогична радио-кнопкам, но каждый вариант оценивается независимо. Фидбэк появляется при проставлении галочки и плавно исчезает при её снятии.

### Открытый вопрос (Ввод текста)
*   Структура: Поле `<input type="text">` и кнопка `.check-input-btn`.
*   Логика:
    *   Верный ответ хранится в `data-correct-answer` у инпута.
    *   При клике на проверку, введенный пользователем ответ приводится к нижнему регистру (toLowerCase) и очищается от пробелов по краям (trim).
    *   Результат выводится в общем блоке `.feedback-area` под вопросом. Тексты обратной связи берутся из атрибутов кнопки `data-feedback` (для верного) и `data-wrong` (для неверного).

## 7. Интерактивная песочница (Python Sandbox)

В модуль интегрирован полноценный редактор кода и интерпретатор Python, работающий в WebAssembly в контексте браузера.

### Внешние зависимости
Загружаются с CDN:
1.  **Pyodide (v0.23.4):** Порт CPython для WebAssembly. Позволяет исполнять Python код без сервера.
2.  **Monaco Editor (v0.38.0):** Редактор кода (основа VS Code). Обеспечивает подсветку, автодополнение, нумерацию строк. Загружается через AMD loader (`require.js`).

### Жизненный цикл песочницы (`script.js`)
1.  **Лоадер:** Пока Pyodide и Monaco скачиваются (десятки мегабайт), вместо редактора показывается CSS-спиннер.
2.  **Инициализация Monaco:**
    *   Для блоков `.sandbox-body` создается инстанс редактора.
    *   Исходный код берется из `data-code` (поддерживается экранирование `\n`).
    *   Высота редактора динамически адаптируется под количество строк (событие `onDidContentSizeChange`).
3.  **Исполнение кода (Интеграция с Pyodide):**
    *   При клике "Запустить", кнопка блокируется.
    *   Создается изолированное пространство имен (`namespace = pyodideInstance.globals.get("dict")()`).
    *   **Перехват `print()`:** В Python внедряется кастомная функция `__custom_print`, которая передает строки в JS функцию `__js_print`, создающую DOM-узлы (текстовые ноды) в блоке вывода. Ошибки передаются в `__js_err_print` и красятся в красный.
    *   **Перехват `input()`:** Поскольку браузерный JS однопоточный, а `input()` синхронно блокирует поток в Python, используется **AST-трансформация** кода перед запуском. Класс `AsyncInputTransformer` на лету заменяет вызовы `input(...)` на `await __async_input(...)`.
    *   JS-функция `__async_input` создает поле `<input>` прямо в области вывода, ожидает нажатия `Enter` (через Promise), и возвращает значение обратно в Python, удаляя инпут из DOM.
4.  **Обработка ошибок:** Синтаксические ошибки Python или ошибки Pyodide перехватываются блоком `catch` в промисе и выводятся в окно вывода красным текстом. После завершения (успех/ошибка) кнопка "Запустить" снова активируется, а namespace удаляется для очистки памяти.

### Ограничения
*   Загрузка среды может занимать время (особенно при медленном интернете) из-за веса Pyodide.
*   При отсутствии интернета загрузка Monaco и Pyodide упадет, лоадер заменится на сообщение об ошибке. Выполнение кода будет невозможно.
